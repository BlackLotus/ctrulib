14 /**
15  * @brief Creates a new libctru thread.
16  * @param entrypoint The function that will be called first upon thread creation
17  * @param arg The argument passed to @p entrypoint
18  * @param stack_size The size of the stack that will be allocated for the thread (will be rounded to a multiple of 8 bytes)
19  * @param prio Low values gives the thread higher priority.
20  * For userland apps, this has to be within the range [0x18;0x3F].
21  * The main thread usually has a priority of 0x30, but not always. Use svcGetThreadPriority() if you need
22  * to create a thread with a priority that is explicitly greater or smaller than that of the main thread.
23  * @param affinity The ID of the processor the thread should be ran on. Processor IDs are labeled starting from 0.
24  * On Old3DS it must be <2, and on New3DS it must be <4.
25  * Pass -1 to execute the thread on all CPUs and -2 to execute the thread on the default CPU (read from the Exheader).
26  * @param detached When set to true, the thread is automatically freed when it finishes.
27  * @return The libctru thread handle on success, NULL on failure.
28  *
29  * - Processor #0 is the application core. It is always possible to create a thread on this core.
30  * - Processor #1 is the system core. If APT_SetAppCpuTimeLimit is used, it is possible to create a single thread on this core.
31  * - Processor #2 is New3DS exclusive. Normal applications can create threads on this core if the exheader kernel flags bitmask has 0x2000 set.
32  * - Processor #3 is New3DS exclusive. Normal applications cannot create threads on this core.
33  * - Processes in the BASE memory region can always create threads on processors #2 and #3.
34  *
35  * @note Default exit code of a thread is 0.
36  * @warning @ref svcExitThread should never be called from the thread, use @ref threadExit instead.
37  */
38 Thread threadCreate(ThreadFunc entrypoint, void* arg, size_t stack_size, int prio, int affinity, bool detached);
